# PowerShell

This section focuses on writing practical PowerShell in a way that is maintainable, testable, and ready for use in team repositories. It is designed to move from basic scripting habits to production-friendly patterns used by VDS.

You will practise taking an idea, turning it into a clear script, and improving it through structure, validation, and safe execution patterns.

## Why This Matters

PowerShell is the entry point for a lot of operational automation. Strong foundations here make every later section easier, especially Pester testing, Nerdio as Code workflows, and reusable CI/CD pipelines.

## Extensions

1. **PowerShell** (ms-vscode.powershell) - syntax support, IntelliSense, formatting, script analysis.
2. **GitLens** (eamodio.gitlens) - useful for understanding history while learning script evolution.

## Learning Goals

- Use clear variable naming and predictable script structure.
- Add parameter validation and defensive checks.
- Separate script logic into reusable functions.
- Use standard output and error handling patterns.
- Build scripts that are easy to review and test.
