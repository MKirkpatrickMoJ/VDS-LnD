# Pester

This section introduces **Pester**, the PowerShell testing framework used to validate script behaviour before code is merged or deployed. The goal is to build confidence that script changes do what you expect and fail safely when they should.

You will learn how to create repeatable tests that cover expected results, edge cases, and failure paths.

## Why This Matters

Testing is one of the fastest ways to improve reliability and reduce regressions. In team environments, tests also make reviews easier because the intended behaviour is documented in executable form.

## Extensions

1. **Pester Test** (pspester.pester-test) - test discovery and execution support.
2. **PowerShell** (ms-vscode.powershell) - test authoring and debugging support.
3. **GitLens** (eamodio.gitlens) - helps track why tests changed over time.

## Learning Goals

- Understand `Describe`, `Context`, and `It` structure.
- Write readable assertions with `Should`.
- Mock external calls to keep tests isolated.
- Cover happy paths and expected failures.
- Use tests to guide refactoring safely.
