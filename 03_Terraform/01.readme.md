# Terraform

This section introduces **Terraform** for infrastructure as code (IaC). You will learn how to define infrastructure declaratively, parameterise it with variables, and manage lifecycle changes in a controlled, auditable way.

The focus is on understanding the Terraform workflow end-to-end: initialise, plan, apply, verify, and destroy.

## Why This Matters

Terraform is a core tool for creating repeatable infrastructure. It reduces manual drift, improves collaboration, and makes changes easier to review before they are applied.

## Setup

Install Terraform with Chocolatey:
`choco install terraform -y`
Verify installation:
`terraform version`

## Extensions

1. **HashiCorp Terraform** (hashicorp.terraform) - core Terraform language support and validation.
2. **Azure Terraform** (ms-azuretools.vscode-azureterraform) - Azure-focused Terraform assistance.
3. **Terraform Graph** (saramorillon.terraform-graph) - visualise resource dependencies.
4. **Terraform** (4ops.terraform) - additional Terraform editing support.

## Learning Goals

- Create and use providers, resources, and variables.
- Separate configuration into sensible files (`main.tf`, `variables.tf`, `outputs.tf`).
- Use plans to inspect changes before apply.
- Understand state basics and why it matters.
- Build and remove a simple environment safely.
