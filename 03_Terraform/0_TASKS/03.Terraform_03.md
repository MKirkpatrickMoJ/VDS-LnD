## Scenario

You should now have code that can deploy a resource group and a workspace.
If your workspace module is still empty, complete Task 2 first.

The remaining resources to define are:
- 2 host pools (one Personal/Persistent and one Pooled/BreadthFirst)
- 2 application groups
- 2 application group assignments

This task focuses on creating the **two host pools**.
Use the [Terraform Registry entry for Host Pools](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/virtual_desktop_host_pool) as your reference.

The previous engineer started this work, but left the host pool variables commented out.

## Task

1. Review the `host-pools` module.
    - It expects a `map(object(...))`, not separate string inputs for each host pool.
    - It uses `for_each`, so one module call can create multiple host pools.
    - `resource_group_name` and `location` are passed separately.
    - The module itself does **not** need to be changed.
2. In the root `main.tf`, add a call to the `host-pools` module.
    - Follow the same pattern as the `resource-groups` and `workspaces` module calls.
    - Pass `host_pools` from `var.host_pools`.
    - Pass `resource_group_name` and `location` from module outputs, in the same way as in task 2.
    - Add `depends_on` so host pools are created only after the resource group is ready.
3. In `vars-dev.tfvars`, uncomment and complete the `host_pools` map.
    - Match the object shape expected by the module.
    - Define both host pools:
      - Pooled host pool: `type = "Pooled"`, `load_balancer_type = "BreadthFirst"`
      - Personal host pool: `type = "Personal"`, `load_balancer_type = "Persistent"`
4. In the root `variables.tf`, declare `host_pools` using `map(object({ ... }))`.
    - This should align with the type used in `modules/host-pools/variables.tf`.
5. Run the essential Terraform commands:
    - `terraform fmt -recursive`
    - `terraform validate`
    - `terraform plan -var-file="./vars-dev.tfvars"`
6. Confirm the plan shows:
    - 1 new resource group
    - 1 new workspace
    - 2 new host pools
    - Values sourced from `vars-dev.tfvars`
