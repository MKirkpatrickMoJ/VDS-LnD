## Scenario

The DEV environment is now created and defined in code. This means the code is available for reuse, one of the many benefits of defining infrastructure as code.

## Task

There is an empty `vars-test.tfvars` file. To create the test environment, all that is required is to populate this `.tfvars` file.

Ensure to capture all the required variables and provide appropriate values to each.
There will be no requirement to add anything to **any** file, other than **vars-test.tfvars**

Once populated, run the usual Terraform commands:
```
terraform fmt -recursive
terraform validate
terraform plan -var-file="./vars-test.tfvars"
```

Note that the plan now points to the `test` variant rather than the `dev` version of the `.tfvars` file. You are welcome to `apply` and `destroy` this version of the code as well if so desired.

## Note

There is also the possibility to create workspaces in Terraform, where code remains very similar but separate `.tfstate` files are maintained per workspace, which can provide more options for managing complex environments. This is not included in this module, but for more reading consult the [official documentation](https://developer.hashicorp.com/terraform/cli/workspaces).
