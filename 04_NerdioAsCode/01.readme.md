# Nerdio as Code

This section bridges operational Nerdio knowledge with code-based delivery. While many tasks still use PowerShell under the hood, the focus here is on applying engineering practices to Nerdio-related automation so changes are repeatable, peer-reviewed, and auditable.

You will practise writing and structuring automation that fits into repository workflows rather than one-off local execution.

## Why This Matters

Moving from GUI-first operations to as-code ways of working improves consistency, change control, and team collaboration. It also makes automation easier to test and easier to run safely across environments.

## Extensions

1. **PowerShell** (ms-vscode.powershell) - primary authoring support.
2. **Pester Test** (pspester.pester-test) - test support for script validation.
3. **YAML** (redhat.vscode-yaml) - useful when automation includes YAML configuration.
4. **GitLens** (eamodio.gitlens) - supports review and collaboration.

## Learning Goals

- Translate operational tasks into scripted workflows.
- Use parameters/config variables instead of embedded values.
- Add logging and safe error handling.
- Create tests for key logic branches.
- Prepare automation for future pipeline execution.
