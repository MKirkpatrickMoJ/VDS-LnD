## Scenario

A colleague started a GHA but had to leave before completing the next step. That next action needs to happen in a new step rather than being added to an existing one. The aim is to find out some basic information about the host running the GHA.

## Task

1) Navigate to `./.github/workflows` and take a look at: `02-task02.yml`.
    - Note that this is a valid starting point and follows the same pattern as `01-task01.yml`.
2) Add an additional step after `Task02 Info Script` to run the command `Get-Host` and write the runner OS, `$env:RUNNER_OS`, to the host.
    - This should also be `shell: pwsh`.
    - This should also be an inline script.
3) Once the additional step has been added to the GHA YAML file, commit the file change and push it to the repository.
    - This can be done within VS Code. Navigate to the `Source Control` tab on the left-hand side.
    - Stage the change to the `02-task02.yml` file using the `+` button next to it.
    - Enter a commit message.
    - Press the dropdown next to the `Commit` button and select `Commit & Push`.
        - If the working branch has not yet been published, you will be prompted to do so before proceeding.
    - If all goes well, the changes to the file will now be available in the repository.
4) Navigate to GitHub and go to the `Actions` tab.
    - Select `LnD-GHA-Task02`.
    - In the `Run workflow` dropdown, change the branch to your working branch (e.g. `NAME-VDS-LnD`).
    - Run the workflow from your branch.
        - Note that if you accidentally run it from `main`, it will run in its original form, not the new version containing your changes.
