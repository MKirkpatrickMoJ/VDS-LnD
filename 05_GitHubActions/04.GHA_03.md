## Scenario

You have already run inline PowerShell within GitHub Actions (GHA). The next step is to run an existing script from a workflow and pass parameters to it at runtime.

A simple PowerShell script for basic network commands has been prepared at `./05_GitHubActions/gha_scripts/03paramScript.ps1`. Your task is to update an existing workflow, found at `./.github/workflows/03-task03.yml` so that the script can be run manually from GitHub Actions with the correct inputs.

## Task

1. Open the PowerShell script at `./05_GitHubActions/gha_scripts/03paramScript.ps1`.
    - Identify the parameters it accepts.
    - Make a note of what the script will do for each action.
2. Open the workflow file for this task at `./.github/workflows/03-task03.yml`.
    - This workflow already uses `workflow_dispatch` inputs.
    - GitHub Actions supports several input types for manually triggered workflows. The [official documentation](https://docs.github.com/en/actions/reference/workflows-and-actions/workflow-syntax#onworkflow_dispatchinputs) explains these in more detail.
    - For this task, the three most relevant input types are:
        - `choice` for selecting from a fixed list of options
        - `boolean` for a true/false value
        - `string` for free-text values such as hostnames or URLs
    - You can view the current inputs in GitHub on the `Actions` tab under `LnD-GHA-Task03`.
3. Update the workflow inputs so that they match the needs of `03paramScript.ps1`.
    - Choose appropriate input types based on the script parameters.
4. Add a new step to the job to call the script and pass the input values to it.
    - Use `shell: pwsh`.
    - Call the script from within the `run: |` block.
    - Reference workflow inputs by using `${{ inputs.NAME_OF_INPUT }}`.
    - Before passing the `DryRun` value to the script, convert it from a workflow input into a PowerShell boolean, for example:
        - `$DryRun = [System.Convert]::ToBoolean("${{ inputs.NAME_OF_YOUR_BOOLEAN_INPUT_HERE }}")`
    - After that, call the script by its file path and pass the corresponding parameter values.
    - This only works because the repository is already checked out earlier in the workflow.
5. Commit and push the updated workflow, then run it from your branch.
    - Check that the inputs appear correctly in GitHub.
    - Run the workflow with test values and confirm that the expected action is taken.
